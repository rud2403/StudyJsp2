비밀번호 :

페이지 전달방식(get/post)

get 방식 / get 메서드

get 방식 = 전송할 데이터(파라미터)를 URL의 정보에 붙여서 전송한다.
장점 : body가 필요없고, 데이터의 길이 제약이 존재하기 때문에 빠른 전송속도
단점 : 1. 데이터(파라미터)의 길이 제약이 존재한다.(256byte 초과 불가)
2. 데이터(파라미터)가 주소줄(URL)에 모두 보인다.(보안적으로 취약)
적은 양의 데이터(파라미터), 일반적인 데이터를 전달 할 때 사용한다.

post 방식 / post 메서드

post 방식 =전송할 데이터(파라미터)를 URL에 전달 하지않고, 본문의 body에 붙여서 전송
장점 : 1. 파라미터가 주소줄(URL)에 전달하지 않는다. -> 보안적으로 우수
2. 파라미터의 길이 제약이 없다. -> 파일업로드
단점 : body 생성, 많은 양의 데이터를 전달하기 때문에 상대적으로 처리속도가 느림
로그인 할 때 post방식을 사용한다.
